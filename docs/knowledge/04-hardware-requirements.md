# ハードウェア要件とGPU選定

## 概要

Gemma 4 27B を使った図面コネクタ検出システムにおける GPU の要件、制約、および選定指針をまとめます。

---

## 1. Gemma 4 27B の VRAM 要件

### パラメータ数とメモリの関係

Gemma 4 27B (270億パラメータ) のメモリ使用量は精度によって大きく変わります。

| 精度 | パラメータあたりのバイト数 | モデルウェイト | アクティベーション込み | 目安 VRAM 合計 |
|------|--------------------------|--------------|----------------------|----------------|
| fp32 | 4 bytes | ~108 GB | +10-20 GB | ~120 GB 以上 |
| fp16 / bf16 | 2 bytes | ~54 GB | +5-10 GB | ~60 GB 以上 |
| int8 (8-bit) | 1 byte | ~27 GB | +3-5 GB | ~30 GB 以上 |
| **int4 (4-bit)** | 0.5 bytes | ~13.5 GB | +2-4 GB | **~15-18 GB** |

> **備考**: アクティベーションサイズはシーケンス長・バッチサイズにより変動します。長いコンテキスト (8K+ トークン) では追加で数GB必要になる場合があります。

## 2. RTX 5090 (32GB VRAM) での評価

### スペック概要

| 項目 | 値 |
|------|----|
| GPU | NVIDIA GeForce RTX 5090 |
| アーキテクチャ | Blackwell (sm_120) |
| VRAM | 32 GB GDDR7 |
| メモリ帯域幅 | ~1,792 GB/s |
| FP16 演算性能 | ~209 TFLOPS |

### 用途別評価

| 用途 | 可否 | 推奨設定 | 備考 |
|------|------|----------|------|
| 推論 (fp16) | 不可 | - | VRAM 不足 (~54GB 必要) |
| **推論 (4-bit AWQ/GPTQ/GGUF)** | **可** | **推奨** | ~15-17GB、余裕あり |
| 推論 (8-bit) | 条件付き可 | 非推奨 | ~30GB、OOM リスク高 |
| LoRA ファインチューニング (QLoRA) | 可 | 推奨 | 4-bit base + fp16 adapter |
| LoRA ファインチューニング (fp16) | 不可 | - | VRAM 不足 |
| フルファインチューニング | 不可 | - | 複数GPU必須 |
| バッチ推論 (batch=1) | 可 | - | 4-bit で実用的 |
| バッチ推論 (batch=4以上) | 制限あり | - | シーケンス長依存 |

### 推論スループット目安 (RTX 5090, 4-bit)

| シーケンス長 | バッチサイズ | トークン/秒 (目安) |
|-------------|-------------|-------------------|
| 512 tokens | 1 | 30-50 tok/s |
| 512 tokens | 2 | 50-80 tok/s |
| 2048 tokens | 1 | 20-35 tok/s |
| 8192 tokens | 1 | 10-20 tok/s |

> **注意**: 上記はあくまで目安です。実際の値は実装・バックエンド・入力内容によって異なります。

## 3. 96GB VRAM マシンでの評価

### スペック例

96GB VRAM を実現する代表的な構成:

| 構成 | VRAM |
|------|------|
| NVIDIA H100 SXM5 (単体) | 80 GB |
| NVIDIA A100 80GB (単体) | 80 GB |
| NVIDIA RTX 6000 Ada × 2 | 48 GB × 2 = 96 GB |
| NVIDIA RTX 5090 × 3 | 32 GB × 3 = 96 GB |
| NVIDIA H100 NVL | 94 GB |

### 用途別評価

| 用途 | 可否 | 推奨設定 | 備考 |
|------|------|----------|------|
| **推論 (fp16/bf16)** | **可** | **推奨** | フル精度、最高品質 |
| 推論 (4-bit) | 可 | - | fp16 が可能なため不要 |
| **LoRA ファインチューニング (fp16)** | **可** | **推奨** | 量子化不要、安定 |
| フルファインチューニング (fp16) | 条件付き可 | - | 27B: ~180GB 必要、勾配チェックポイント併用で可 |
| **大規模バッチ処理** | **可** | **強み** | バッチサイズ 8-16 が現実的 |
| **複数モデル同時ロード** | **可** | **強み** | 7B + 27B の同時展開も可能 |
| マルチモーダル処理 | 可 | - | 画像エンコーダ分の追加 VRAM も確保可能 |

### 推論スループット目安 (96GB, fp16)

| シーケンス長 | バッチサイズ | トークン/秒 (目安) |
|-------------|-------------|-------------------|
| 512 tokens | 1 | 40-60 tok/s |
| 512 tokens | 8 | 150-250 tok/s |
| 2048 tokens | 4 | 100-180 tok/s |
| 8192 tokens | 2 | 50-100 tok/s |

## 4. 96GB vs 32GB 業務効率比較

### 総合比較表

| 評価軸 | RTX 5090 (32GB) | 96GB VRAM マシン | 差分 |
|--------|-----------------|-----------------|------|
| 量子化の必要性 | 4-bit 必須 | 不要 (fp16 可) | 品質差あり |
| 推論精度 | 量子化誤差あり | フル精度 | 96GB が有利 |
| 推論スループット (単体) | 基準 (1x) | **2〜4x** 向上見込み | バッチサイズ増による |
| ファインチューニング方式 | QLoRA のみ | LoRA (fp16) / フル | 96GB が柔軟 |
| ファインチューニング時間 | 基準 (1x) | **2〜3x** 短縮見込み | バッチサイズ・精度向上 |
| 実験サイクル速度 | 遅め | **速い** | 試行錯誤が容易 |
| 複数モデル同時運用 | 困難 | 容易 | 96GB が有利 |
| 初期コスト | 低〜中 | 高 | 32GB が有利 |

### 推論速度の差について

96GB マシンでは fp16 + 大バッチ処理が可能なため:

- **バッチサイズ 8** での処理が可能 → スループットで **2〜4x** 向上
- メモリ帯域幅の活用率が高まる
- KV キャッシュをより多く保持できるため、長文処理で特に有利

### ファインチューニング時間の差について

96GB マシンでは:

- fp16 LoRA が使えるため QLoRA より収束が安定・速い
- より大きいバッチサイズ (4〜8) → 同じ epoch でも更新回数が異なる
- 勾配蓄積が少なくて済む → 実効的な学習ステップが増える

結果として同じデータセットに対して **2〜3x** の時間短縮が見込めます。

### 検出精度への影響

| 要因 | 32GB (4-bit) | 96GB (fp16) |
|------|-------------|-------------|
| モデル重みの精度 | 4-bit 量子化 | フル fp16 |
| 量子化誤差 | あり | なし |
| コネクタ種別の識別精度 | 良好〜普通 | 最良 |
| 複雑な図面での性能 | やや低下の可能性 | 安定 |

## 5. GPU 選定の考え方とコスト感

### 用途別推奨

| プロジェクトフェーズ | 推奨 GPU | 理由 |
|----------------------|----------|------|
| PoC・検証フェーズ | **RTX 5090 (32GB)** | コスト低、4-bit で十分動く |
| 本番運用（高スループット必要） | **96GB VRAM 構成** | 品質・速度ともに上 |
| ファインチューニング重視 | **96GB VRAM 構成** | fp16 LoRA、速い実験サイクル |
| 低予算・個人開発 | **RTX 5090 (32GB)** | 現実的な選択肢 |

### コスト感（2025年時点の概算）

| ハードウェア | 価格帯 (概算) | 備考 |
|-------------|-------------|------|
| RTX 5090 (32GB) | 30〜35万円 | コンシューマー向け最上位 |
| RTX 6000 Ada (48GB) × 2 | 60〜80万円 | プロフェッショナル向け |
| NVIDIA A100 80GB (中古) | 60〜100万円 | データセンター向け、入手困難な場合あり |
| NVIDIA H100 80GB | 300〜500万円 | 最新ハイエンド |

### 結論

- **今すぐ始める・コストを抑える**: RTX 5090 + 4-bit 量子化で十分に実用的
- **精度・スループットを最大化**: 96GB VRAM 構成を検討
- **段階的アプローチ**: RTX 5090 で PoC → 効果が確認できたら 96GB 構成に移行

## 6. メモリ効率化のヒント

32GB 環境での VRAM 節約テクニック:

```python
# 1. gradient checkpointing（学習時）
model.gradient_checkpointing_enable()

# 2. Flash Attention 2 の利用
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    attn_implementation="flash_attention_2",
    torch_dtype=torch.float16,
)

# 3. 不要なキャッシュを随時クリア
import torch
torch.cuda.empty_cache()

# 4. 最大シーケンス長の制限
# vLLM の場合
python -m vllm.entrypoints.openai.api_server \
  --max-model-len 4096 \  # デフォルト 8192 を制限
  --gpu-memory-utilization 0.85  # VRAM の 85% を上限に
```
