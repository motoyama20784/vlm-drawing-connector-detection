# VLM（Vision Language Model）基礎知識

## VLMとは何か

VLM（Vision Language Model）は、**テキストと画像の両方を入力として処理できる大規模言語モデル**である。従来のLLM（Large Language Model）がテキストのみを扱うのに対し、VLMは視覚情報と言語情報を統合的に理解・生成できる。

代表的なモデル: GPT-4V、Gemini、LLaVA、Gemma 4、Qwen-VL、InternVL など

---

## アーキテクチャの概要

VLMは主に以下の3つのコンポーネントで構成される。

### 1. Vision Encoder（ViT: Vision Transformer）

- 入力画像を固定サイズのパッチ（例: 14×14ピクセル）に分割する
- 各パッチをベクトルに変換し、位置エンコーディングを付加する
- Transformer の Self-Attention により、パッチ間の関係を学習する
- 出力は「視覚トークン列」（画像を表す埋め込みベクトルの系列）

```
画像 → パッチ分割 → Linear Projection → Position Encoding → Transformer Blocks → 視覚トークン列
```

### 2. Language Model（Transformer Decoder）

- テキストトークンと視覚トークンを統合して処理する
- 自己回帰的（auto-regressive）にテキストを生成する
- GPT系のアーキテクチャが主流

### 3. Cross-modal Projection（視覚-言語ブリッジ）

- Vision Encoder が出力する視覚トークンは、Language Model の入力空間と次元・分布が異なる
- Projection 層（MLP または Linear Layer）が視覚トークンをテキスト空間にマッピングする
- この層のアライメント学習がVLMの品質に大きく影響する

```
Vision Encoder → [視覚トークン] → Cross-modal Projection → [言語空間トークン] → Language Model → テキスト出力
```

## 従来の画像認識（CNN/YOLO）との違い

| 特性 | CNNベース（YOLO等） | VLM |
|---|---|---|
| タスク定義 | 学習時に固定（クラス数・種類） | プロンプトで動的に指定可能 |
| 出力形式 | 固定（クラスID + bounding box） | 柔軟（JSON, テキスト, 説明文など） |
| ゼロショット性能 | 低い（学習クラス外は検出不可） | 高い（未知クラスも言語で記述可能） |
| 推論速度 | 非常に高速 | 比較的低速 |
| 必要データ量 | タスクごとに大量のアノテーション | プロンプトのみで動作可能 |
| 解釈性 | 低い | 高い（説明文として出力可能） |

### CNNベースモデルの動作原理

```
画像入力 → CNN特徴抽出 → [特定クラス専用の最終層] → クラス確率 / bounding box
```

- 検出クラスは学習時に決定される
- 「コネクタ」「端子」「プラグ」などを別々のクラスとして定義・学習が必要
- 新しいクラスを追加するには再学習が必要

### VLMの動作原理

```
画像 + テキスト指示 → VLM → 自然言語による回答
```

- タスクをプロンプトで柔軟に切り替えられる
- 「コネクタを検出して」「部品番号を読み取って」「図面の接続関係を説明して」などを同一モデルで実行可能

## VLMで「最終層を変えてタスク切り替え」はできるか？

**結論: NO**

CNNベースのモデルでは、学習済みの特徴抽出部分（backbone）はそのままに、最終分類層（head）を差し替えることで異なるタスクに転用できる（Transfer Learning）。

しかしVLMはこのアーキテクチャとは根本的に異なる。

### なぜVLMで最終層の差し替えが機能しないか

- VLMは **Generative Model（生成モデル）** である
- 出力はクラス確率ベクトルではなく、**トークン系列**（テキスト）として生成される
- 最終層はLM Head（語彙サイズのsoftmax）であり、これを差し替えてもVLMの本質的な動作は変わらない

### VLMでのタスク切り替え方法

#### 方法1: プロンプトによる指示（最もシンプル）

同一モデルで異なるタスクを実行できる。

```
# コネクタ検出タスク
「この図面の中にあるコネクタをすべて検出し、位置を[x1,y1,x2,y2]形式のJSONで出力してください」

# OCRタスク
「この図面に記載されている部品番号をすべて読み取ってください」

# 説明タスク
「この電気図面の接続構成を日本語で説明してください」
```

#### 方法2: ファインチューニング（タスク特化）

特定タスクの精度を向上させたい場合は、LoRA等でファインチューニングを行う（詳細は `06-fine-tuning.md` を参照）。

## マルチモーダルモデルの学習フェーズ

VLMの学習は通常、以下の段階で構成される。

### フェーズ1: Vision Encoder の事前学習

- 大量の画像データで ViT を事前学習する
- CLIP（Contrastive Language-Image Pretraining）等の手法で画像-テキストペアを用いた対照学習を行う
- この段階で視覚的な特徴表現（色、形状、テクスチャ、物体など）を獲得する

### フェーズ2: Projection 層のアライメント学習

- Vision Encoder と Language Model を Cross-modal Projection で接続する
- 画像-テキストペアを用いて、視覚トークンを言語空間にマッピングする方法を学習する
- 多くの場合、Vision Encoder と Language Model は **凍結**し、Projection 層のみを学習する

### フェーズ3: Instruction Tuning（SFT: Supervised Fine-Tuning）

- 「画像 + 指示文 → 期待される回答」の形式のデータセットで学習する
- モデルが指示に従って回答を生成できるようにする
- 例: LLaVA-Instruct, ShareGPT4V 等のデータセットを使用

## VLMを使った物体検知のアプローチ

### アプローチ1: Zero-shot（ゼロショット）

学習データなしに、プロンプトだけで検出を依頼する。

```
「次の図面画像を解析し、電気コネクタをすべて特定してください。
各コネクタの位置を画像の左上を原点(0,0)とした座標で[x1,y1,x2,y2]形式で示してください。
回答はJSON形式で出力してください。」
```

- **メリット**: データ収集・学習不要、即座に試せる
- **デメリット**: 精度が限定的、座標の正確性に欠ける場合がある

### アプローチ2: Few-shot（フューショット）

プロンプト内にサンプル画像と正解例を与えて誘導する。

```
「以下の例を参考に、図面からコネクタを検出してください。
例1: [サンプル画像の説明] → {"connectors": [{"bbox": [100, 200, 150, 250]}]}
例2: [サンプル画像の説明] → {"connectors": [{"bbox": [300, 100, 380, 160]}]}
では、この図面を解析してください: [対象画像]」
```

- **メリット**: 少量のサンプルで精度向上
- **デメリット**: コンテキスト長に制約がある

### アプローチ3: ファインチューニング

タスク特化データでVLMを学習させる。

- アノテーション済みの図面データで学習
- LoRA / QLoRA を用いてリソース効率よく実施可能
- **メリット**: 高精度、一貫した出力形式
- **デメリット**: データ収集・学習コストが発生する

## VLM学習に必要な知識

VLMを深く理解・応用するために以下の知識が必要である。

### 基礎理論

- **Transformer / Attention の理解**
  - Self-Attention の計算方法（Q, K, V行列）
  - Multi-head Attention
  - Positional Encoding

- **ViT (Vision Transformer) の仕組み**
  - パッチ化（Patchification）の概念
  - [CLS] トークンの役割
  - ImageNet での事前学習

- **Contrastive Learning（対照学習）**
  - CLIP の学習原理（正例ペアのコサイン類似度を最大化、負例を最小化）
  - 視覚-言語の対応関係の学習方法

### 学習手法

- **Instruction Tuning / RLHF の基礎**
  - SFT（Supervised Fine-Tuning）の概念
  - RLHF（Reinforcement Learning from Human Feedback）の概念
  - DPO（Direct Preference Optimization）

### 実装・運用

- **データセット形式（画像-テキストペア）**
  - 画像 + 指示文 + 回答のペア形式
  - アノテーションツール（Label Studio 等）の使い方

- **分散学習の基礎（大規模モデルの場合）**
  - データ並列、モデル並列の概念
  - DeepSpeed / FSDP の基礎
  - 勾配チェックポイント（gradient checkpointing）
