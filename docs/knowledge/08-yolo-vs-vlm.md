# YOLO vs VLM 比較（コネクタ検出への適用）

## YOLOとは

- You Only Look Once: リアルタイム物体検知の代表的モデル
- CNN ベースの検出器、bounding box + クラス確率を直接出力
- YOLOv8 / YOLOv10 / YOLOv11 などが現行版

---

## YOLO のメリット（VLM比較）

- **推論速度**: RTX 5090 で数ms/枚（VLMは数秒〜数十秒）
- **精度**: 十分な学習データがあればmAP90%以上も可能
- **軽量**: モデルサイズ数MB〜数十MB（VLMは数十GB）
- **実装の簡潔さ**: `ultralytics` ライブラリで数行
- **学習が安定**: 確立された手法、ノウハウが豊富
- **エッジデバイスにも展開可能**

---

## YOLO のデメリット（VLM比較）

- **アノテーションデータが必須**: bounding boxのラベリング作業が必要（数百〜数千枚）
- **固定クラス**: 学習時に定義したクラスしか検出できない
- **文脈理解なし**: 「断面Aのコネクタのみ」のような条件付き検出は苦手
- **転用困難**: 新しいコネクタ種類が増えるたびに再学習が必要
- **説明生成不可**: 検出したものの説明・補足情報を出せない

---

## VLM のメリット（YOLO比較）

- **Zero-shot対応**: ラベルなしで即座に試せる
- **柔軟な指示**: 「断面Aの端子だけ」「赤いコネクタのみ」など自然言語で指定
- **説明・属性生成**: 検出物の種類・接続先・状態を説明できる
- **新クラス対応**: 見たことのないコネクタも文脈から推測
- **ドキュメント連携**: 図面の注記やテキストと組み合わせた理解

---

## VLM のデメリット（YOLO比較）

- **低速**: 1枚あたり数秒〜数十秒（バッチ処理に不向き）
- **重い**: モデルが数十GB、専用GPU必須
- **座標精度**: bounding box の精度はYOLO未満（ファインチューニング後でも）
- **コスト**: 推論コスト高（電力・時間）
- **安定性**: プロンプトの微妙な変化で出力が変わることがある

---

## どちらを選ぶか（判断基準）

| 状況 | 推奨 |
|------|------|
| アノテーション済みデータが大量にある | YOLO |
| 高速処理が必要（リアルタイム/バッチ） | YOLO |
| データがなくゼロショットで試したい | VLM |
| コネクタの種類・説明も必要 | VLM |
| 条件付き検出（「断面Aのみ」等） | VLM |
| 最高精度が必要 | YOLO（データあり）またはVLM ファインチューニング |

---

## ハイブリッドアプローチ（推奨）

1. YOLO でコネクタ候補を高速に検出
2. 検出領域を VLM に渡して詳細分類・属性抽出

→ 速度と柔軟性を両立

---

## YOLOの推論精度について

- **十分なラベル済みデータ（500枚以上）があれば YOLO の方が精度が高い**
- ゼロショット VLM の精度: mAP で 30-60% 程度（ドメイン依存）
- 学習済み YOLO の精度: mAP 80-95% が現実的
- ただし VLM はファインチューニング後に YOLO に近い精度を出せることもある
