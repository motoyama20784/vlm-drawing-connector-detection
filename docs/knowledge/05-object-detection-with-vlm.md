# VLMによる物体検知の適用方法

## VLMで物体検知をする3つのアプローチ

### アプローチ1: プロンプトエンジニアリング（Zero-shot / Few-shot）

**学習データ不要で即座に試せる**最も手軽なアプローチ。

#### Zero-shot（ゼロショット）

```
「この図面の中にあるコネクタ（接続端子）をすべて見つけ、
各コネクタの位置を画像の左上を原点(0,0)としたピクセル座標で
[x1, y1, x2, y2]形式で出力してください。
回答は以下のJSON形式のみで返してください:
{
  "connectors": [
    {"id": 1, "bbox": [x1, y1, x2, y2]}
  ]
}」
```

- **メリット**: データ収集・学習コスト不要、即座にプロトタイピング可能
- **デメリット**: 精度が限定的（50〜70%程度）、座標精度が低い傾向がある

#### Few-shot（フューショット）

プロンプト内に例示を含めることで精度を向上させる。

```
「以下の例を参考に、コネクタを検出してJSON形式で出力してください。

[例1]
入力: (サンプル図面1の説明や特徴)
出力: {"connectors": [{"id": 1, "type": "terminal", "bbox": [120, 340, 165, 390]}]}

[例2]
入力: (サンプル図面2の説明や特徴)
出力: {"connectors": [{"id": 1, "type": "plug", "bbox": [80, 100, 130, 145]},
                       {"id": 2, "type": "terminal", "bbox": [220, 300, 270, 355]}]}

では、添付の図面を解析してください。」
```

- **メリット**: ゼロショットより高精度、データ学習不要
- **デメリット**: コンテキスト長の制限（サンプル数に上限あり）、推論コスト増加

### アプローチ2: Grounding + VLM（2段階パイプライン）

**専用の物体検出器（Grounding モデル）とVLMを組み合わせる**ハイブリッドアプローチ。

#### パイプライン構成

```
入力画像
    ↓
[Stage 1] Grounding DINO / SAM（Segment Anything Model）
    ↓ 候補領域（bounding box / マスク）の抽出
[Stage 2] 各候補領域をクロップ → Gemma 4 へ入力
    ↓ 領域ごとの分類・説明・部品番号の読み取り
[最終出力] 検出結果の統合
```

#### 主要な Grounding モデル

| モデル | 特性 |
|---|---|
| Grounding DINO | テキスト指定による open-vocabulary 検出 |
| SAM 2 | セグメンテーション特化、任意物体のマスク生成 |
| OWL-ViT | ゼロショット物体検出、軽量 |

#### 実装例（概念コード）

```python
# Stage 1: Grounding DINO でコネクタ候補領域を検出
candidate_boxes = grounding_dino.detect(
    image=drawing_image,
    text_prompt="connector . terminal . plug . socket"
)

# Stage 2: 各候補をVLMで詳細分類
results = []
for box in candidate_boxes:
    crop = drawing_image.crop(box)
    response = gemma4.generate(
        image=crop,
        prompt="この部品の種類と部品番号を特定してください。JSON形式で回答してください。"
    )
    results.append({"bbox": box, "details": parse_json(response)})
```

- **メリット**: Stage 1 で高速に候補を絞り込み、Stage 2 でVLMの豊かな言語能力を活用
- **デメリット**: Stage 1 の精度に依存する、2段階処理によるレイテンシ増加

### アプローチ3: ファインチューニング

**VLMにアノテーション済みデータを学習させ**、タスク特化のモデルを構築するアプローチ。

```
アノテーション済み図面データセット
（画像 + コネクタ位置 + 種類のラベル）
    ↓
LoRA / QLoRA ファインチューニング
    ↓
コネクタ検出特化 Gemma 4 モデル
    ↓
高精度・一貫した出力形式での検出
```

- **メリット**: 高精度（85〜95%）、一貫した出力形式、ドメイン固有シンボルへの対応
- **デメリット**: データ収集・アノテーションコスト、学習時間（数時間〜数日）

詳細は `06-fine-tuning.md` を参照。

## 図面（技術図面・回路図）への適用の注意点

### ドメイン固有性の高さ

一般的な写真・画像とは異なり、技術図面は以下の特性を持つ。

| 特性 | 影響 |
|---|---|
| シンボルが抽象的・ドメイン固有 | 一般学習では認識が困難 |
| 線画・モノクロが多い | テクスチャ・色彩情報が少ない |
| 高密度な情報 | 小さなシンボルが多数存在 |
| 統一された描画規約（JIS, IEC等） | 規格を知らないと誤解釈 |

### Zero-shot での問題点

ドメイン特化学習なしの状態では以下の問題が発生しやすい。

- **見落とし（偽陰性）**: 図面固有のコネクタシンボルを認識できない
- **誤検出（偽陽性）**: コネクタ以外の記号をコネクタと誤認する
- **座標精度の低下**: 精密な bounding box が得られない
- **出力形式のばらつき**: プロンプトを破って説明文を混入する

### 推奨対応策

1. **高解像度入力**: 図面は600DPI以上でスキャン・入力する
2. **画像分割処理**: 大きな図面はタイル状に分割してVLMに入力する
3. **プロンプトの厳密化**: 出力スキーマを明示し、余分なテキストを禁止する
4. **ファインチューニング**: ドメイン固有データで学習させる（最終的に必須）
5. **後処理の追加**: NMS（Non-Maximum Suppression）等で重複検出を除去する

## 出力フォーマットの設計

### 推奨 JSON スキーマ

```json
{
  "connectors": [
    {
      "id": 1,
      "type": "terminal",
      "bbox": [120, 340, 165, 390],
      "confidence": 0.9,
      "part_number": "CN1",
      "description": "電源入力コネクタ、2ピン"
    },
    {
      "id": 2,
      "type": "plug",
      "bbox": [280, 150, 330, 200],
      "confidence": 0.85,
      "part_number": "J2",
      "description": "信号出力プラグ、4ピン"
    }
  ],
  "total_count": 2,
  "image_size": {"width": 1920, "height": 1080}
}
```

### フィールド定義

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | int | コネクタの通し番号（1始まり） |
| `type` | string | コネクタ種類（terminal / plug / socket / header / other） |
| `bbox` | array[4] | 座標 [x1, y1, x2, y2]（左上原点、ピクセル単位） |
| `confidence` | float | 検出信頼度（0.0〜1.0） |
| `part_number` | string | 図面上の部品番号（不明の場合は null） |
| `description` | string | コネクタの詳細説明（オプション） |

### プロンプト内でのスキーマ指定例

```
「添付の図面を解析し、電気コネクタを検出してください。
回答は以下のJSONスキーマに**厳密に**従ってください。
説明文・前置き・コードブロック記号は一切含めないでください。

{
  "connectors": [
    {
      "id": <整数>,
      "type": "<terminal|plug|socket|header|other>",
      "bbox": [<x1_int>, <y1_int>, <x2_int>, <y2_int>],
      "confidence": <float 0.0-1.0>,
      "part_number": "<文字列またはnull>"
    }
  ],
  "total_count": <整数>
}」
```

## VLMとYOLOのハイブリッドアプローチ

精度と推論速度を両立するための実用的な構成。

### アーキテクチャ

```
入力図面画像
    ↓
[高速検出層] YOLOv11 / YOLOv8（ファインチューニング済み）
    ↓ 高速でコネクタ候補 bounding box を検出（数十ms）
[詳細解析層] Gemma 4 (VLM)
    ↓ 各候補領域を受け取り、種類・部品番号・詳細を出力（数百ms〜数秒）
[統合・後処理]
    ↓
最終検出結果（座標 + 種類 + 詳細情報）
```

### 役割分担

| コンポーネント | 役割 | 特性 |
|---|---|---|
| YOLO | 候補領域の高速検出 | 高速（リアルタイム可能）、学習が必要 |
| Gemma 4 | 詳細分類・説明・OCR | 高精度・柔軟、推論に時間がかかる |

### メリット

- **速度**: YOLO が候補を絞るためVLMへの入力を最小化できる
- **精度**: VLMがコネクタの種類・部品番号を正確に読み取る
- **スケーラビリティ**: YOLO部分を差し替えやすい設計

### デメリット

- YOLO のファインチューニングも別途必要
- パイプラインが複雑になる
- YOLO の見落としはVLMでもカバーできない

### 実装コスト比較

| アプローチ | 実装コスト | 期待精度 | 推論速度 |
|---|---|---|---|
| Zero-shot VLM のみ | 低（即日） | 50〜70% | 遅い（秒単位） |
| Grounding + VLM | 中（数日） | 70〜85% | 中（秒単位） |
| YOLO + VLM ハイブリッド | 高（数週間） | 85〜95% | 速い（百ms単位） |
| VLM ファインチューニング | 高（数週間） | 85〜95% | 遅い（秒単位） |
| YOLO のみ | 中（1〜2週間） | 80〜90% | 非常に速い（ms単位） |

**本プロジェクトの推奨方針**: まずZero-shot VLMでプロトタイプ検証 → ファインチューニングで精度向上 → 要件に応じてYOLOハイブリッドを検討。
